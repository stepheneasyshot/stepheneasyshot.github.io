# 【AI】旅行规划Agent系统搭建
AI智能体的风吹遍了整个技术圈。在看完基础理论框架文档后，我坐不住了——正所谓“纸上得来终觉浅，绝知此事要躬行”，我决定亲自下场，动手搭建一套属于自己的 **AI Agent 助理系统**。

大语言模型（LLM）就像是一个拥有无穷知识的“超级大脑”，但要让它真正走进现实、解决复杂问题，还必须为它装上“四肢”和“眼睛”——也就是搭建外围的工具链（Tools）和基础设施。这也是 Agent 最迷人、最具有交付价值的地方。

**缘起：当客户端开发者遇上完美旅行**

作为一名深耕**客户端开发**的程序猿，在日常生活中，我习惯了从用户的视角去审视各种 App 的交互与体验。最近，我迷上了一款旅行规划类 App，经常用它来给自己定制各种特立独行的旅行攻略。

在惊叹于其出色的产品逻辑之余，作为开发者的职业病也犯了：**“如果用现在的 AI Agent 架构，我自己能不能把它的核心功能复刻出来？”**

这个念头一旦萌生便再也按捺不住。旅行规划是一个典型的、需要多步骤推理的复杂场景——它需要实时联网查机票酒店、需要根据预算和偏好做路线排布、还需要在计划冲突时进行动态调整。这简直是绝佳的 Agent 练兵场！

接下来我将以一个客户端开发者的视角，完整记录这个旅行助手 Agent 从零到一的架构设计与代码敲定过程。

**⚠️ 免责声明**

本项目的开发完全基于公开功能进行**逆向需求推导与架构实现**。所有核心逻辑、Prompt 结构及后端服务均为本人独立设计开发，无任何商业目的。如有雷同，纯属巧合。

## 技术库选取

### 1. LLM 与 Agent 框架层（大脑与神经中枢）

这部分库是 Agent 的核心，负责处理大模型的调用以及复杂任务流的编排。

* **`langchain-core`**
* **一句话简介**：LangChain 生态的基石。
* **博客大白话**：它是整个 LangChain 框架的“普通话”。它定义了什么是 Prompt、什么是 LLM 输入输出、什么是工具（Tools）。有了它，后续切换不同大模型或组件时，代码才能保持高度一致。


* **`langchain-openai`** 与 **`langchain-ollama`**
* **一句话简介**：云端顶配与本地私有化大模型的完美结合。
* **博客大白话**：这是我的 Agent 的“左右脑”。`langchain-openai` 负责调用云端的 GPT 系列模型，处理复杂推理；而 `langchain-ollama` 则让我能够无缝接入本地部署的开源大模型（如 Llama3、DeepSeek），既省钱又保护隐私。


* **`langgraph`**
* **一句话简介**：让 Agent 告别“死脑筋”，实现复杂的循环工作流。
* **博客大白话**：传统的 LangChain 链条是线性的（A接着B）。但真实的 Agent 需要反复思考、出错重试。`langgraph` 把任务变成了“图（Graph）”结构，让我的 Agent 拥有了**循环状态管理**的能力，真正实现了类似“思考 -> 行动 -> 观察 -> 再思考”的智能循环。



### 2. 工具与数据获取层（眼睛与双手）

Agent 不能只靠空想，它需要通过工具去感知外部世界。

* **`mcpcn-web-search-mcp`**
* **一句话简介**：基于 MCP 协议的联网搜索利器。
* **博客大白话**：这是项目的一大亮点！它是一个遵循 **MCP（Model Context Protocol，模型上下文协议）** 的联网搜索工具，让 Agent 能够实时检索互联网上的最新信息，彻底解决大模型的知识幻觉问题。


* **`requests`** 与 **`httpx`**
* **一句话简介**：网络请求的“双子星”。
* **博客大白话**：Agent 要和外界通信，网络请求必不可少。`requests` 依旧是处理常规同步请求的黄金标准；而 `httpx` 则带来了强大的**异步（async/await）支持**，在需要并发请求多个 API 时，性能直接拉满。


* **`beautifulsoup4`** 与 **`lxml`**
* **一句话简介**：网页数据提取的“手术刀”。
* **博客大白话**：当 Agent 联网搜索抓回一堆杂乱的 HTML 网页时，这两位就派上用场了。`beautifulsoup4` 提供了极其人性化的 API 来解析网页，而 `lxml` 作为它的底层加速引擎，保证了海量文本内容解析时的极速响应。



### 3. 数据校验与基础架构层（骨骼与免疫系统）

确保 Agent 处理的数据是干净、健壮且结构化的。

* **`pydantic`**
* **一句话简介**：全自动的数据校验与结构化神器。
* **博客大白话**：大模型的输出往往是不可控的字符串，怎么把它们变成代码里可以直接用的对象？全靠 `pydantic`。它帮我定义了严密的数据模型，不仅能自动校验 LLM 输出的结构，还能把 Agent 的 Tool 输入约束得严丝合缝。



### 4. API 服务与实时通信层（对外通道）

把写好的 Agent 包装成服务，供前端或其他系统调用。

* **`fastapi`** 与 **`uvicorn[standard]`**
* **一句话简介**：高性能的异步 Web API 框架。
* **博客大白话**：Agent 开发完了，得开放接口给外界用。`fastapi` 绝对是目前 Python 圈的顶流，写起来快，自带 Swagger 文档；配合 `uvicorn` 这个高性能的 ASGI 服务器，轻松扛住 Agent 的异步并发请求。


* **`sse-starlette`**
* **一句话简介**：实现 ChatGPT 同款“打字机”流式效果的关键。
* **博客大白话**：看大模型一个字一个字往外蹦的体验真的很爽。这个库让我的 FastAPI 服务支持 **SSE（Server-Sent Events，服务器发送事件）**，从而能把 Agent 的思考过程和生成内容，像流水一样实时推送到前端。
